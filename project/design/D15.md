# Decision 15 — Versioning, version reporting, and release distribution

**Decision.** agentrepl's version is owned by the git tag, never by a source
literal. `cmd/agentrepl/main.go` declares a single linker seam:

```go
// cmd/agentrepl/main.go
var version = "dev" // overridden at build time via -ldflags "-X main.version=..."
```

**Reporting (`-V` / `--version`).** The composition root prescans argv before
`repl.ParseArgs`, exactly as it already does for `-h` (`hasHelpFlag`). A parallel
`hasVersionFlag` matches `-V`, `-version`, and `--version`; when present, `run`
writes `version` plus a trailing newline to stdout and returns `0` — the REPL loop
never starts, and a bad `-c` (or any other flag) on the same line is not consulted
(version short-circuits, mirroring help). The value is emitted verbatim: `run`
never parses or reformats it. Because `run` is the tested seam, an in-process call
with `-V` on the unstamped test binary yields exactly `dev\n`.

```go
// cmd/agentrepl/main.go
func hasVersionFlag(args []string) bool // true for "-V", "-version", "--version"
```

Both `-V` and `--version` are surfaced in the launch flag table (Decision 4 lists
the row) and in the self-describing `--help` catalog's `flags:` block
(`internal/repl`'s `WriteHelp`), which gains a `-V, --version   show the version
and exit` line. Version reporting is credential- and config-blind: like `-h`, it
builds no `config.Target` and reads no environment.

**Stamping.** Two builders inject the version through the same
`-ldflags -X main.version=` seam:

- **`make build` / `make install`** stamp local builds from git state:

  ```
  go build -ldflags "-X main.version=$(shell git describe --tags --always --dirty)" -o $(BIN_DIR)/$(BINARY) ./cmd/agentrepl
  ```

  A clean semver tag reports the tag (`v0.1.0`); a commit past a tag reports the
  describe form (`v0.1.0-3-gabc1234`); modified sources append `-dirty`; a build
  with no ldflags (every `go test` binary) reports `dev`.

- **goreleaser** stamps release builds from the pushed tag
  (`-X main.version={{ .Tag }}`).

**Release (`.goreleaser.yaml` + `.github/workflows/release.yml`).** Cutting a
release is pushing a semver tag; no version literal is edited. `.goreleaser.yaml`
(goreleaser v2) declares:

- `project_name: agentrepl`; one build, `main: ./cmd/agentrepl`,
  `binary: agentrepl`, `CGO_ENABLED=0`, `goos: [linux, darwin]`,
  `goarch: [amd64, arm64]`, `ldflags: -s -w -X main.version={{ .Tag }}`.
- `archives`: `tar.gz`, **versionless** asset names
  `{{ .Binary }}_{{ .Os }}_{{ .Arch }}` so `install.sh` can hit a stable
  `releases/latest/download/…` URL with no API call.
- `checksum` → `checksums.txt`; `release.github` owner `ikigenba`, name
  `agentrepl`; `changelog.use: github`.

`.github/workflows/release.yml` triggers `on: push: tags: ['v*']`, grants
`contents: write`, checks out with `fetch-depth: 0` (goreleaser's changelog needs
full history), sets up Go 1.26, and runs `goreleaser/goreleaser-action@v6` with
`args: release --clean` and `GITHUB_TOKEN`.

**Install (`install.sh`).** A POSIX `#!/bin/sh` script fetches a prebuilt release
binary onto the PATH, no Go toolchain required:

```sh
curl -fsSL https://raw.githubusercontent.com/ikigenba/agentrepl/main/install.sh | sh
```

- `REPO=ikigenba/agentrepl`, `BINARY=agentrepl`.
- `BINDIR` (default `${PREFIX:-$HOME/.local}/bin`), `PREFIX`, and
  `AGENTREPL_VERSION` (default `latest`, else a `vMAJOR.MINOR.PATCH` tag) are the
  only knobs.
- Detects `uname -s`/`-m` → `linux|darwin` × `amd64|arm64`; an unsupported
  OS/arch exits non-zero with a message.
- Fetches `releases/latest/download/${asset}` (or
  `releases/download/${AGENTREPL_VERSION}/${asset}`) where
  `asset=${BINARY}_${os}_${arch}.tar.gz`, untars, `install -m 0755` into `BINDIR`,
  and warns if `BINDIR` is not on `PATH`.

`v0.1.0` is the first tag to push; it is **not** written into any source, config,
or doc as a maintained constant — it appears only as an illustrative pin example.

**README.** The install section gains the one-line `curl … | sh` install (latest
by default, `AGENTREPL_VERSION=v0.1.0` to pin, `BINDIR`/`PREFIX` to relocate)
alongside the existing `make build`/`make install` from-source path.

**Rejected.**

- **A hand-maintained version constant in source** — a second source of truth that
  drifts from the tag; the linker seam makes git the only owner, matching how the
  agentkit pin is treated as a plan/build fact rather than a doc literal.
- **`runtime/debug.ReadBuildInfo` fallback** — a second, less predictable version
  path for a binary that is always built via the Makefile or goreleaser; `dev` is
  an honest, explicit answer for an unstamped build.
- **Registering `V`/`version` as `flag.FlagSet` bools** — the composition-root
  prescan mirrors the existing `-h` handling, keeps version reporting credential-
  and config-blind, and lets it short-circuit ahead of `-c` validation exactly as
  help does.
- **Versioned release asset names** — force an API call to discover the latest
  tag; versionless names give `install.sh` a stable `latest/download` URL.
- **A separate release Makefile target or a tag-writing script** — a release is
  `git tag vX.Y.Z && git push --tags`; the workflow does the rest.
- **A bash installer piped to `bash`** — the script uses only POSIX `sh`, so
  `#!/bin/sh` piped to `sh` is more portable and matches the sibling `embed`
  installer it mirrors.

**Verification.**

- R-S45L-UT0N — an unstamped build (no ldflags; the `go test` binary) invoking
  `run` with `-V` writes exactly `dev` + newline to stdout, returns exit code `0`,
  and never starts the REPL loop.
- R-S5DI-8KRC — building the real `cmd/agentrepl` package with
  `go build -ldflags "-X main.version=<sentinel>"` and executing the produced
  binary with `-V` writes exactly `<sentinel>` + newline to stdout; this exercises
  the real linker seam, not the in-process default.
- R-S6LE-MCI1 — invoking `run` with `--version` (and with `-version`) behaves
  identically to `-V`: exactly `dev` + newline on the unstamped binary, exit
  code `0`.
- R-S7TB-048Q — the self-describing `--help` catalog's `flags:` block names both
  `-V` and `--version`.
