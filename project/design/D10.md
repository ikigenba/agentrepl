# Decision 10 — Standard tools via agentkit's toolkit

**Decision.** agentrepl ships no tools of its own. The conversation's tools come from agentkit's `toolkit` subpackage (agentkit ≥ v0.7.0): the composition root resolves the process working directory once (`os.Getwd()` in `cmd/agentrepl/main.go`) and the REPL wires

```go
Tools: toolkit.All(cwd),
```

which yields the six standard coding tools in toolkit's conventional order — `Bash`, `Read`, `Write`, `Edit`, `Glob`, `Grep`. The former `internal/tools` package is deleted; its behavior (and its tests) now live upstream in agentkit.

- **Root = process cwd, resolved at the composition root.** `main.go` calls `os.Getwd()` and passes the result down; a `Getwd` failure is a startup failure (exit 1, per the spine's exit-code taxonomy). No config key selects the root — the harness is launched from wherever the tools should be rooted.
- **Toolkit semantics are adopted wholesale, as shipped.** File tools confine paths beneath the root (symlink-aware; an accident guard, not a security sandbox); `Bash` runs with the root as its working directory and preserves combined output on non-zero exit; `Edit` requires a unique exact match; results are capped at 30k characters; `Bash` has timeout handling. agentrepl neither wraps, filters, nor re-verifies any of this — proving toolkit behavior is agentkit's suite's job; agentrepl proves only its own wiring.
- **Division of proof.** agentrepl's Verification covers exactly what agentrepl can get wrong: that the six toolkit tools (and nothing else) are offered, and that the root passed is the process cwd. Tool semantics carry no agentrepl ids.

**Rejected.**
- **Keeping `internal/tools`** — the toolkit exists precisely to end per-consumer reimplementations of bash/read/write/edit; agentrepl's job is to exercise agentkit as shipped, and a homegrown parallel set would demonstrate the wrong thing.
- **A `tools.root` config key** — a knob nobody asked for; launching from the desired directory already selects the root.
- **Re-proving toolkit semantics (confinement, unique-match edit, output cap) in agentrepl's suite** — duplicates agentkit's own tests and breaks on every upstream change.
- **Preserving the old lowercase names / replace-all edit semantics via wrappers** — agentrepl adopts the shipped surface verbatim; wrapping contradicts the thin-consumer principle.

**Verification.**
- R-W3MV-WRJ7 — the conversation is configured with exactly the six toolkit tools, in toolkit's conventional order, named `Bash`, `Read`, `Write`, `Edit`, `Glob`, `Grep` — no more, no fewer, no others.
- R-W4US-AJ9W — the tools are rooted at the process working directory: invoking a wired file tool (e.g. `Read` or `Write`) with a relative path from a test whose working directory is a temp dir resolves against that temp dir, proving the root passed was the cwd and not a fixed or empty path.
