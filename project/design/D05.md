# Decision 5 ‚Äî Turn execution, the Renderer, and color

**Decision.** The turn driver (in `repl`) owns the loop; the `Renderer` (in `render`) owns presentation ‚Äî a genuine two-impl interface, the one place runtime polymorphism is warranted.

```go
package render

// Renderer presents the input prompt, turn events, outcome, and spend.
type Renderer interface {
    Prompt()                                                     // draw the input prompt before a read (decorated+TTY: "you ‚Ä∫ "; raw/non-TTY: no-op)
    Input(text string)                                           // the operator's entered turn message (raw records it; decorated does not echo it)
    Event(ev agentkit.Event)                                     // each turn event, in order
    Usage(turn agentkit.Usage, turnCost, total agentkit.Cost)    // per-turn usage/cost line (raw only; decorated no-op)
    Summary(total agentkit.Usage, totalCost agentkit.Cost)       // cumulative usage+cost block (/usage, on exit)
    Warning(w agentkit.Warning)                                  // a setting agentkit could not honor as asked
    Error(err error)                                             // a failed turn or command
    Notice(line string)                                          // agentrepl info (e.g. /dump, hints)
}

func NewDecorated(out io.Writer, color, tty bool) Renderer
func NewRaw(out io.Writer) Renderer
```

`Prompt()` and `Input(text)` split what was one `Prompt(text)` method into the two jobs it conflated: drawing the input affordance (before a read, no text, terminal-only) and recording the entered message (after a read, raw's forensic stream). The decorated view never echoes the operator's input ‚Äî the terminal already shows what was typed at the prompt; raw never draws an affordance ‚Äî it has no interactive surface. The decorated renderer is constructed with both `color` and `tty`: `tty` gates whether the `you ‚Ä∫` prompt is drawn at all (an interactive terminal echoes typed input so the line reads `you ‚Ä∫ hi`), independent of `color`, which only gates ANSI (so `NO_COLOR=1` in a terminal still gets an uncolored prompt). `/render` reconstructs from `state.io.IsTTY`.

**Input affordance is loop-owned, not driver-owned.** The `Run` loop (Decision 9) calls `Renderer.Prompt()` *before* awaiting each input line ‚Äî so the `you ‚Ä∫` prompt precedes **every** read uniformly: a turn message, a `/command`, or an empty line. The prompt is the renderer's, not the driver's; the driver no longer draws it.

**Turn driver** (`repl`): pre-check provider+model ‚Üí `Input(text)` (records the message; raw-only effect) ‚Üí `waiter.Start(conv.Model)` (decorated+TTY only ‚Äî the wait status line, Decision 13; `nopWaiter` otherwise) ‚Üí `stream := conv.Send(ctx, text)` ‚Üí range `stream.Events()` calling `Event(ev)` for each, with `waiter.Stop()` on the first event and again via `defer` so every exit (success, error, empty, interrupt) erases the line ‚Üí after draining, if `ctx.Err() != nil` render an interrupt notice and exit (Decision 6); otherwise **relay any settings warnings first** ‚Äî `for _, w := range stream.Warnings() { Warning(w) }` ‚Äî then, if `stream.Err() != nil` call `Error(err)`, else call `Usage(stream.Usage(), stream.Cost(), conv.TotalCost())`. The driver calls `Usage` uniformly; the decorated renderer no-ops it (per-turn spend is raw-only ‚Äî Decision 7), so the difference lives in the renderer, not the driver. Warnings are rendered whether the turn then succeeds or errors, because they describe a setting that was not honored (most often reasoning: a non-native value carried in via the Decision 3 carve-out, which agentkit warned-and-defaulted), independent of turn outcome. The `ctx` passed to `Send` is the SIGINT-cancellable context from Decision 6; the loop survives any non-interrupt turn error.

**Warning relay is render-only.** agentrepl never mints, classifies, or suppresses a warning ‚Äî it forwards each `agentkit.Warning` verbatim to the renderer. A `Warning` is **not** an `Error` (the turn was issued and, for the reasoning case, succeeds with the model's default); it gets its own kind so its treatment and placement (before the usage line) are distinct.

**decorated** (default): a distinct visual treatment per kind ‚Äî `Prompt()` draws the `you ‚Ä∫` input prompt (**bold**, default foreground ‚Äî the terminal's own white; ANSI emitted only when `color`; no trailing newline, so the terminal's own echo of typed input completes the line, and that echoed input keeps the default foreground too) **only when `tty`**, and draws nothing otherwise; `Input(text)` is a **no-op** (the entered text is never re-rendered ‚Äî the terminal already showed it); a `MessageDone` is rendered by walking `ev.Message.Blocks` **in order** and emitting one decorated block per content block ‚Äî a `ReasoningBlock` (non-empty `Summary`) as a **dim** `reasoning ‚Ä∫` block; a `TextBlock` (non-empty `Text`) as an `assistant ‚Ä∫` block whose label is **bold + light blue** (ANSI bright-blue) and whose reply text is the **same light blue but not bold** (the hue carries the whole reply; only the label is bold); and a `ToolUseBlock` as a **subdued gray** `tool call ‚Ä∫` line (ANSI bright-black, label and body alike) ‚Äî so a turn's reasoning, visible text, and tool calls render **interleaved in exactly the order the model emitted them**, and a block whose content is empty emits nothing. The standalone `ToolUse` event is a **no-op** in decorated: the tool call is already rendered from the message's `ToolUseBlock`, so it is never doubled. A `ToolResult` event (the tool result is fed back as a separate message, never inside a `MessageDone`) renders as a **subdued gray** line when successful and in the **red** error treatment when `IsError`, so a failing tool stands out rather than blending into the gray. `Warning` in a distinct warn style (e.g. `‚öÝ <Setting>: <Detail>`, separate from the error style); `Usage(...)` is a **no-op** (per-turn spend is raw-only ‚Äî Decision 7). Delivery is **message-granular**: each block prints whole, with its own trailing newline ‚Äî there is no token-level streaming and no open streamed line to flush ‚Äî and **no `‚îÄ` separator rule** is printed between messages (the prior turn/message separator is removed). Exact glyphs/labels/ANSI codes are pinned by golden files under `testdata/`.

**Vertical spacing ‚Äî one blank line between consecutive blocks, as a leading separator.** The decorated view separates every adjacent pair of transcript blocks (the `you ‚Ä∫` prompt, an `assistant ‚Ä∫` reply, `reasoning ‚Ä∫`, `tool call ‚Ä∫`, `tool result ‚Ä∫`, a notice, a warning, an error, the summary) with exactly **one blank line**, for a uniform vertical rhythm. The blank is emitted as a **leading** separator: a single newline written at the *start* of each block, suppressed until the first block of the session has been drawn ‚Äî so the session opens with no leading blank and closes with no trailing blank after the final summary, while the blank the operator perceives "after the `you ‚Ä∫` line" is the leading separator of the block that follows it. The `you ‚Ä∫` prompt therefore never owns a trailing newline (the terminal's echo of the typed line completes it); whatever block comes next ‚Äî a reply, a tool call, or a command's notice ‚Äî supplies the separator, so turns, `/commands`, and notices space identically. A **bare empty line** at the prompt (ignored per Decision 9) produces **no** extra blank: the separator appears only between actual blocks, never between two consecutive prompts. To keep the single-blank rule exact regardless of model or tool output, the `reasoning ‚Ä∫` / `assistant ‚Ä∫` / `tool call ‚Ä∫` / `tool result ‚Ä∫` emitters **trim a single trailing newline** from the rendered summary/text/input/output, so a block whose content ends in `\n` still yields exactly one blank, not two.

**raw**: emits **one undecorated JSON line per committed entry** ‚Äî the prompt entry from `Input(text)` (`{"type":"prompt","text":‚Ä¶}`), `MessageDone`, `ToolUse`, `ToolResult` ‚Äî plus the per-turn usage line as JSON and **one JSON line per `Warning`** (the `agentkit.Warning` struct marshaled verbatim, carrying `Setting`/`Code`/`Detail`); `Prompt()` is a **no-op** (no interactive affordance in raw); there are no token-delta events to skip ‚Äî `MessageDone.Message` already carries the fully assembled reasoning/text/tool-use blocks. Raw still records the prompt text and the per-turn usage, and ‚Äî unlike decorated ‚Äî it emits the standalone `ToolUse` event as its own JSON entry (one line per entry is the raw contract, so the tool call appears both inside the `MessageDone` message and as a `ToolUse` entry). Never emits ANSI. Marshals with `encoding/json`, yielding block shapes consistent with agentkit's own log (agentkit does no custom marshaling).

**color**: the composition root computes `color = IO.IsTTY && Getenv("NO_COLOR") == ""` and passes it **plus `IO.IsTTY`** to `NewDecorated(out, color, tty)`. The prompt is drawn when `tty` (an interactive terminal); ANSI is emitted when `color`. Raw is always colorless and draws no prompt.

**Rejected.**
- **Raw = tee agentkit's `Conversation.Log` (LogRecord stream) to stdout** ‚Äî byte-identical to the forensic file, but it makes `rawRenderer` a no-op while output sneaks in via a `MultiWriter`, turning the two-impl seam into a fiction and coupling raw render to the log. Marshaling events keeps both renderers honest and the log independent.
- **Decorated renders tool calls from the standalone `ToolUse` event** (not the message's `ToolUseBlock`) ‚Äî rejected: agentkit yields the whole `MessageDone` *before* any `ToolUse` event, so this forces all of a message's text ahead of all its tool calls and destroys the model's true reasoning/text/tool-call interleaving. Rendering from the message blocks in order, with the `ToolUse` event a decorated no-op, preserves the real order without doubling.
- **Surfacing token-level deltas to the consumer** ‚Äî agentkit (Phase 24, message-granular delivery) no longer exposes `TextDelta`/`ReasoningDelta`; the smallest unit is a completed `MessageDone`. Decorated renders whole blocks and leans on the wait status line (Decision 13) to cover pre-message latency.
- **A `Renderer` per event-kind / a visitor** ‚Äî overwrought; a small sealed-union `switch` inside each impl is the idiomatic Go shape.

**Verification.**
- R-LL9K-SKDQ ‚Äî decorated renders each transcript kind ‚Äî reply text and reasoning summary (from `MessageDone.Message`), tool-call (from a `ToolUseBlock`), and tool-result (from a `ToolResult` event) ‚Äî with a distinct treatment, and prints **no** `‚îÄ` separator rule between messages (golden).
- R-Q52T-PXCR ‚Äî decorated separates every adjacent pair of transcript blocks with exactly one blank line (leading separator): a multi-block transcript (`you ‚Ä∫` ‚Üí reply ‚Üí tool call ‚Üí tool result ‚Üí reply) shows one blank line between each pair, no leading blank before the first block, no trailing blank after the summary, a tool result whose output ends in `\n` still yields exactly one blank, and a bare empty line at the prompt adds no blank (golden).
- R-JFBW-TYU8 ‚Äî the loop calls `Renderer.Prompt()` before every input read (turn, command, and empty line alike); in decorated mode `Prompt()` writes the `you ‚Ä∫` prompt (no trailing newline) when `tty` and writes nothing when not a `tty`, and `Input(text)` never echoes the entered text back as a transcript line (golden for the tty/non-tty decorated output; repl-level assertion that the prompt precedes each read).
- R-JGJT-7QKX ‚Äî in decorated mode `Usage(...)` prints nothing (per-turn spend suppressed); the same driver call in raw mode emits the per-turn usage line.
- R-CCHP-S6AJ ‚Äî decorated renders a `MessageDone` by walking `Message.Blocks` in order, emitting a dim `reasoning ‚Ä∫` block for each non-empty `ReasoningBlock.Summary`, an `assistant ‚Ä∫` block for each non-empty `TextBlock`, and a gray `tool call ‚Ä∫` line for each `ToolUseBlock`, so reasoning, visible text, and tool calls appear interleaved in the model's emitted order; a block with empty content emits nothing, and no token-level streaming occurs (golden).
- R-CDPM-5Y18 ‚Äî decorated treats the standalone `ToolUse` event as a no-op so a tool-calling message renders each `tool call ‚Ä∫` exactly once (no doubling against the `ToolUseBlock` already rendered from `MessageDone`), while the `ToolResult` event still renders a tool-result line; raw, by contrast, emits the `ToolUse` event as its own JSON entry.
- R-LNPD-K3V4 ‚Äî decorated emits ANSI color when `color` is true and none when it is false (golden for both).
- R-OBNM-N6XX ‚Äî decorated applies the fixed palette (golden): the `you ‚Ä∫` prompt and its echoed input are bold/default-foreground (no hue); the `assistant ‚Ä∫` label is bold + light blue and the reply text is the same light blue but not bold; `tool call ‚Ä∫` and a successful `tool result ‚Ä∫` line are subdued gray (label and body); a `tool result ‚Ä∫` with `IsError` is rendered in the red error treatment; `reasoning ‚Ä∫` is dim.
- R-LOX9-XVLT ‚Äî raw emits exactly one undecorated JSON line for the prompt entry (from `Input`), and one per `MessageDone`/`ToolUse`/`ToolResult`, plus a per-turn usage line, and emits nothing for `Prompt()` (there are no delta events); the output is valid JSONL with no ANSI.
- R-LRD2-PF37 ‚Äî a `ToolResult` with `IsError` gets the error treatment in decorated and preserves `IsError` in raw.
- R-LSKZ-36TW ‚Äî on a non-interrupt failed turn the driver calls `Error` (not `Usage`) and the loop continues to the next input.
- R-G480-F0ID ‚Äî after draining a turn's events, the driver calls `Renderer.Warning` once per entry in `stream.Warnings()`, before `Usage`/`Error`, and verbatim (the agentkit `Warning` is forwarded unmodified, never reclassified or suppressed); a turn with no warnings calls `Warning` zero times.
- R-G5FW-SS92 ‚Äî decorated renders a `Warning` in its own warn treatment, distinct from the `Error` treatment (golden); raw emits exactly one JSON line carrying the `Warning`'s `Setting`/`Code`/`Detail`.
- R-G6NT-6JZR ‚Äî a reasoning value the selected model does not natively understand (set via any reasoning key ‚Äî the Decision 3 carve-out) produces an agentkit reasoning warning that the driver relays to the renderer, and the turn still completes with the model's default applied.
