# Decision 3 — Config-key namespace, resolution & coercion

**Decision.** `internal/config` is the single vocabulary that both the launch flag and the runtime slash-command funnel through — an **explicit typed key table** (no reflection), extended with the selection/auth state the v0.4.0 agentkit surface makes the consumer's job: model→provider resolution against agentkit's advisory catalog, per-provider auth-method selection, and **lazy provider construction**.

```go
package config

// Target is what config reads and mutates.
type Target struct {
    Conv     *agentkit.Conversation
    Cat      []catalog.Provider
    Getenv   func(string) string

    ProviderName     string // selected provider name; seeded "openai"
    ProviderExplicit bool   // true once the operator set provider (flag or /set); false for the seed
    ModelName        string // model as the operator entered it; seeded "gpt-5.6-sol"
    Auth             string // "key" | "sub" | "" (unset → the provider's default method)
    AuthFile         string // subscription auth file path; seeded to the default (Decision 14)
    BaseURL          string // base-URL override for the active provider; "" → baked-in default

    ReasoningRaw string // last raw reasoning value, for display (ReasoningValue is opaque); "" = unset
    ReasoningKey string // which native reasoning key set it; "" = unset

    built agentkit.Provider // lazily built; invalidated by provider/auth/auth_file/base_url changes
}

// NewTarget returns a Target seeded with the launch defaults:
// provider=openai (not operator-explicit), model=gpt-5.6-sol (resolved against
// the agentkit catalog, pricing installed), auth unset (→ sub for openai),
// auth_file at its default path. No provider is constructed.
func NewTarget(conv *agentkit.Conversation, cat []catalog.Provider, getenv func(string) string, authFileDefault string) *Target

// Provider returns the built agentkit provider for the current selection,
// constructing it on first use (catalog.Provider.New with the current Options)
// and caching it until a selection/auth/base_url change invalidates it.
// Construction errors (missing key, missing/bad auth file, unsupported auth)
// are returned — the lazy-failure seam Decision 14's directive message renders.
func (t *Target) Provider() (agentkit.Provider, error)

// Set applies one key=value to t. A non-empty notice is advisory output the
// caller renders (today: the free-flow acceptance warning); err is a hard
// rejection that mutated nothing.
func Set(t *Target, key, raw string) (notice string, err error)

func Get(t *Target, key string) (string, bool)
func Dump(t *Target) []string
func Keys() []string
func ParsePair(s string) (key, value string, err error)

var (
    ErrUnknownKey = errors.New("unknown config key")
    ErrBadValue   = errors.New("invalid value for config key")
)
```

The key table (flat, unprefixed):

| Key | Target state | Type |
|-----|--------------|------|
| `provider` | `ProviderName` (+`ProviderExplicit`), invalidates `built` | provider name (table-validated) |
| `model` | `ModelName` → resolution (below) → `Conv.Model`, `Conv.Pricing` | model id |
| `auth` | `Auth`, invalidates `built` | `key` \| `sub` (method-supported check) |
| `auth_file` | `AuthFile`, invalidates `built` | path |
| `base_url` | `BaseURL`, invalidates `built` | URL string |
| `system` | `Conv.System` | string |
| `temperature` / `top_p` | `Gen.Temperature` / `Gen.TopP` | `*float64` |
| `max_tokens` | `Gen.MaxTokens` | int |
| `effort` / `thinking_budget` / `thinking_level` / `thinking` | `Gen.Reasoning` (+`ReasoningRaw`/`ReasoningKey`) | native reasoning (below) |
| `max_attempts` | `Retry.MaxAttempts` | int |
| `base_delay` / `max_delay` / `max_elapsed` | `Retry.*` | duration |
| `ignore_retry_after` | `Retry.IgnoreRetryAfter` | bool |
| `tool_loop_limit` | `Conv.MaxToolIterations` | int |

- **Launch defaults are seeded state, not magic.** `NewTarget` seeds `provider=openai`, `model=gpt-5.6-sol`, `auth` unset (resolving to `sub` for openai), `auth_file` at its default. Bare `agentrepl` is therefore immediately usable (subject to lazy auth, Decision 14); `-c` pairs then apply over the seed in encounter order. The seeded provider is **not** operator-explicit.

- **Model resolution — the two-tier rule.** On `model=<name>`:
  - **Provider not operator-explicit** (the seed, or after `provider=default`): the name **must** resolve in agentkit's catalog — `catalog.Resolve("", name)`. On success the entry's home provider becomes `ProviderName` (still not operator-explicit — it was derived), `Conv.Model` gets the wire model, `Conv.Pricing` gets `entry.Pricing`. An uncataloged name is a hard wrapped `ErrUnknownModel` ("not in the agentkit catalog; set provider explicitly to send it anyway") and mutates nothing.
  - **Provider operator-explicit**: `catalog.Resolve(ProviderName, name)`. Cataloged → wire model (a routed model gets its slug, e.g. `glm-5.2` under `openrouter` → `z-ai/glm-5.2`) + pricing. Uncataloged → **free-flow**: accepted verbatim as `Conv.Model`, `Conv.Pricing` set nil, and `Set` returns a **notice** ("model not in catalog: no pricing (cost reports 0), reasoning unchecked") the caller renders. You named the destination; you own the string.
  - `Get`/`Dump` render `model` as the operator entered it (`ModelName`), not the wire slug.
- **`provider=<name>`** validates against the table (`catalog.Lookup` → wrapped `ErrUnknownProvider` listing the five), stores the name, marks it operator-explicit, and invalidates the built provider. It does **not** construct (construction is lazy) and does not touch the model — a now-mismatched model simply follows the free-flow rule at its next `model=` set, or rides to the wire as-is (agentkit sends it; the provider's verdict surfaces per Decision 11).
- **`auth=<method>`** accepts only `key`/`sub` (`ErrBadValue` otherwise) and rejects, with a clear error naming provider and method, a method the *current* provider's `Methods` does not contain — so `auth=sub` under `zai` fails at set time, loudly, mutating nothing. Unset `Auth` resolves to the provider's default method at build time (openai → `sub`, others → `key`). `auth_file=<path>` stores the path used by the `sub` build.
- **`base_url`** stores an override applied to **whichever provider is active** at build time via `catalog.Options.BaseURL` (Decision 2) — order-independent with `provider`, and cleared by `base_url=default`.
- **Lazy construction & invalidation.** `Target.Provider()` builds via the table entry's `New(getenv, Options{BaseURL, Auth, AuthFile})` on first need, assigns `Conv.Provider`, and caches. Any change to `provider`/`auth`/`auth_file`/`base_url` invalidates the cache so the next turn rebuilds. Construction errors are returned to the caller (the turn driver), never swallowed — Decision 14 owns the directive rendering.
- **Reasoning: key-directed coercion, catalog-gated for cataloged models, pass-through otherwise.** The key name picks the constructor exactly as before (`effort`/`thinking_level` → `Level(verbatim)`; `thinking_budget` → `Budget(int)`, non-integer → `ErrBadValue`; `thinking` → `on`→unset zero / `off`→`DisableReasoning()`, other tokens → `ErrBadValue`). Then, **when the current model is cataloged** (`catalog.Lookup(ModelName)` with a non-nil `Reasoning` spec), the built value is checked with `akcatalog.Check(model, v)`: a rejected value returns a wrapped `ErrBadValue` naming the accepted values from the spec — the gate replaces the retired agentkit-side warn-and-default, which v0.4.0 removed. When the model is **free-flow** (uncataloged), any structurally-parseable value is accepted and passes to the wire unchecked; the provider's verdict surfaces as an ordinary turn error (Decision 11). A cataloged-model switch *after* a reasoning set may leave a stale value; it rides to the wire and surfaces the same way — set-time gating checks only the model current at set time.
- **The `default` sentinel** resets a key to its *seeded* state: `provider=default` → `openai`, explicit flag cleared; `model=default` → `gpt-5.6-sol` re-resolved; `auth=default` → unset (provider default); `auth_file=default` → the default path; every other key → its zero/unset state (nil pointer, zero value, cleared reasoning), rendered as `default` by `Get`/`Dump` exactly as before.
- Both control surfaces share this one entrypoint: `-c` does `ParsePair`+`Set`; `/set` calls `Set`; `/dump` calls `Dump`. Notices render on both paths.

**Rejected.**
- **Eager provider construction at `provider=` time** (the prior design) — with a subscription default and no auth file, bare startup would fail before the operator could create the auth file; laziness is what makes the out-of-box path work, and it also stops paying construction cost for selections never used.
- **Gating free-flow reasoning** — there is no spec to gate against; inventing one contradicts pass-through.
- **Keeping the warn-and-default reasoning carve-out** — its agentkit backing (`WarnReasoningUnsupported` + default fallback) was removed in v0.4.0; without it the "accepted then silently mis-sent" value would surface as a confusing wire error for *cataloged* models whose specs we hold. Set-time `catalog.Check` gives the clear, immediate error the product now promises.
- **`Conv.Model` holding the operator's name and translating at send** — agentkit has no translation hook; the wire model must be on the conversation. Display keeps the operator's name via `ModelName`.
- **A `model!` force syntax for free-flow** — the explicit-provider requirement already encodes intent; a second escape hatch is redundant ceremony.

**Verification.**
- R-LYK7-Y7ZS — every known key coerces its value to the correct typed `Target` field (table-driven across the full key list, including pointer, int, duration, bool, enum, and string kinds).
- R-LZS4-BZQH — an unknown key returns a wrapped `ErrUnknownKey` naming the key and mutates nothing.
- R-4ZNS-WI9Q — an unparseable value returns a wrapped `ErrBadValue` naming the key and reason, and mutates nothing (every key; for the four reasoning keys this covers structural failures — non-integer `thinking_budget`, non-`on`/`off` `thinking`, empty values — while catalog rejection is asserted by R-53BI-1THT).
- R-4X80-4YSC — `NewTarget` seeds the defaults: `provider=openai` (not operator-explicit), `model=gpt-5.6-sol` resolved with its catalog pricing installed on `Conv.Pricing`, `auth` unset, `auth_file` at the supplied default path; `Dump` renders exactly this state, and no provider is constructed by seeding alone.
- R-4YFW-IQJ1 — the `default` sentinel restores the seeded state for `provider` (openai, explicit flag cleared), `model` (gpt-5.6-sol re-resolved), `auth` (unset), and `auth_file` (default path), and resets every other key to zero/unset rendered as `default`.
- R-50VP-AA0F — with provider not operator-explicit, `model=<cataloged>` derives and stores the entry's home provider and installs the wire model + pricing; `model=<uncataloged>` returns a wrapped `ErrUnknownModel` and mutates nothing.
- R-523L-O1R4 — with provider operator-explicit: a cataloged model resolves to its wire form (`glm-5.2` under `openrouter` → `Conv.Model` `z-ai/glm-5.2`) with pricing installed; an uncataloged model is accepted verbatim with `Conv.Pricing` nil and `Set` returning a non-empty free-flow notice; `Get`/`Dump` render the operator-entered name in both cases.
- R-4TKA-ZNK9 — `auth` accepts exactly `key`/`sub` (`ErrBadValue` otherwise); a method outside the current provider's `Methods` (e.g. `auth=sub` under `zai`) is rejected at set time with a clear error naming provider and method, mutating nothing; unset `Auth` resolves at build time to `sub` for openai and `key` for the other four.
- R-4W03-R71N — `auth_file` stores the path and, with `auth` resolving to `sub`, the lazily built openai provider is constructed from that file; setting any of `provider`/`auth`/`auth_file`/`base_url` invalidates the cached provider so the next `Target.Provider()` rebuilds, while an unchanged selection reuses the cached instance.
- R-54JE-FL8I — `Target.Provider()` constructs lazily via the table entry with `Options{BaseURL, Auth, AuthFile}`, assigns `Conv.Provider`, and returns construction errors (missing key, bad auth file, unsupported method) to the caller unswallowed.
- R-53BI-1THT — reasoning gating: with a cataloged model current, a coerced reasoning value rejected by `akcatalog.Check` returns a wrapped `ErrBadValue` naming the spec's accepted values and mutates nothing, while an accepted value lands on `Conv.Gen.Reasoning`; with a free-flow model current, the same non-native value is accepted and stored unchecked.
- R-FZCE-VXJL — reasoning coercion is key-directed: `effort`/`thinking_level` → `Level(verbatim)` (`high`, `xhigh`, `none`); `thinking_budget` → `Budget(n)` for a base-10 integer including negatives (`-1`, `0`, `8000`); `thinking=off` → `DisableReasoning()` and `thinking=on` → the unset zero `ReasoningValue`; in every case the result is assigned to `Conv.Gen.Reasoning` and the raw string + key stored on `ReasoningRaw`/`ReasoningKey`.
- R-G304-18RO — `Get`/`Dump` render reasoning from `ReasoningRaw` under `ReasoningKey`, the other reasoning keys and an unset value as `default`; setting any reasoning key to `default` resets `Conv.Gen.Reasoning` to the zero `ReasoningValue` and clears `ReasoningRaw`/`ReasoningKey`.
- R-M4NP-V2P9 — `Dump` returns all keys sorted as `key=value` lines reflecting current state.
- R-M5VM-8UFY — flag/runtime parity: `ParsePair`+`Set` and a direct `Set` reach identical state (and identical notice/error) for the same key and value.

(The base-URL construction seam itself is verified under Decision 2; this decision's `base_url` ordering/invalidation behavior is covered inside R-4W03-R71N.)
