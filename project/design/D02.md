# Decision 2 — Provider table & agentkit-catalog sourcing

**Decision.** `internal/catalog` shrinks to a **provider table** — the wiring agentkit deliberately does not own: provider names, env keys, credential construction, and supported auth methods. **All model-shaped data comes live from `agentkit/catalog`** (`Lookup`/`Resolve`/`ListByProvider`/`Check`, `Entry.Reasoning`/`Entry.Pricing`): agentrepl curates nothing — agentkit's advisory catalog *is* the list.

```go
package catalog

// AuthMethod names one way a provider authenticates.
type AuthMethod string

const (
    AuthKey AuthMethod = "key" // API key from the provider's env var
    AuthSub AuthMethod = "sub" // OpenAI subscription auth from a raw-token-response auth file
)

// Options carries per-construction overrides threaded from config (Decision 3).
type Options struct {
    BaseURL  string     // override the provider's API root; "" → provider default
    Auth     AuthMethod // "" → the provider's default method (openai: sub; others: key)
    AuthFile string     // path to the subscription auth file (Decision 14); used only when Auth resolves to sub
}

// Provider is one agentkit provider agentrepl can drive.
type Provider struct {
    Name    string       // "anthropic" | "google" | "openai" | "openrouter" | "zai"
    EnvKey  string       // "…_API_KEY" env var for AuthKey
    Methods []AuthMethod // supported auth methods; first entry is the default
    New     func(getenv func(string) string, opts Options) (agentkit.Provider, error)
}

// Default returns the built-in table of the five providers, in a stable order.
func Default() []Provider

// Lookup finds a provider by name.
func Lookup(cat []Provider, name string) (Provider, bool)

// Models returns the chat models agentkit's catalog lists for provider name:
// akcatalog.ListByProvider(name) with embedding entries (Entry.Embedding != nil)
// filtered out, in ListByProvider's sorted order.
func Models(name string) []akcatalog.Entry

// Resolve maps a (provider, model) selection to its wire form via
// akcatalog.Resolve: provider=="" derives the entry's home provider; a routed
// model yields its wire slug (glm-5.2 under openrouter → "z-ai/glm-5.2");
// ok=false means the model is not cataloged for that selection.
func Resolve(provider, model string) (routeProvider, wireModel string, entry akcatalog.Entry, ok bool)

var (
    ErrUnknownProvider = errors.New("unknown provider")
    ErrUnknownModel    = errors.New("unknown model")
    ErrMissingKey      = errors.New("missing API key")
    ErrAuthUnsupported = errors.New("auth method not supported by provider")
)
```

- **The table carries five providers**: `anthropic` (`ANTHROPIC_API_KEY`), `google` (`GEMINI_API_KEY`), `openai` (`OPENAI_API_KEY`), `openrouter` (`OPENROUTER_API_KEY`), `zai` (`ZAI_API_KEY`). `Methods` is `[key]` for all except `openai`, which is `[sub, key]` — subscription first because it is the provider's (and the product's) default.
- **Construction goes through agentkit's credential constructors.** Each entry's `New` builds `pkg.New(pkg.APIKey(getenv(EnvKey)), opts…)` for `AuthKey` — returning a wrapped `ErrMissingKey` naming the env var when the key is empty — and, for `openai`+`AuthSub`, `openai.New(openai.Subscription(store))` where `store` comes from `subscription.Load(opts.AuthFile)` (Decision 14); a `Load` failure is returned wrapped, naming the path. An `opts.Auth` the entry's `Methods` does not contain returns a wrapped `ErrAuthUnsupported` naming both provider and method.
- **`Options.BaseURL` applies to every provider**, mapped to that package's `WithBaseURL(...)` option — the proxy/mock seam a verification harness needs. Empty leaves the baked-in default root. (This generalizes what was previously a Z.ai-only override.)
- **Model data is never stored in agentrepl.** `Models` and `Resolve` are thin adapters over `agentkit/catalog`; the model lists, reasoning specs (`Entry.Reasoning`), pricing (`Entry.Pricing`), and routes are read at call time. `Models` filters embedding entries out — agentrepl's surface is chat-only (embeddings are deliberately unexposed).
- **Free-flow is allowed, resolution is Decision 3's job.** `Resolve` reports `ok=false` for an uncataloged selection without judging it; the config layer decides when that is a hard error (bare model) versus accepted pass-through (explicit provider). This mirrors agentkit's own advisory stance.
- `ErrMissingKey`/`ErrAuthUnsupported` are ordinary non-fatal data; the REPL renders them and stays alive (Decisions 11/14).

**Rejected.**
- **Keeping a curated `Models []string` per provider** — agentkit's catalog now owns the tables; a local mirror is pure drift risk and re-adds the anti-drift test machinery for no gain.
- **A `Catalog` interface** — speculative; only the constructor needs swapping, which the injected `New` func covers.
- **Reading the agentkit catalog once into agentrepl structs at startup** — a cache layer over an in-process data table; call-time reads cannot go stale and need no invalidation.
- **A pseudo-provider `openai-sub`** — auth method is a construction option, not a different provider; a sixth table row would leak auth into the provider namespace and complicate model resolution.

**Verification.**
- R-4IL7-JPW0 — `Default()` returns exactly the five providers `anthropic`, `google`, `openai`, `openrouter`, `zai`, each carrying its contractual env key (`ANTHROPIC_API_KEY`, `GEMINI_API_KEY`, `OPENAI_API_KEY`, `OPENROUTER_API_KEY`, `ZAI_API_KEY`), and `Lookup` reports not-found for an unknown name.
- R-4JT3-XHMP — `Default()` sets `Methods` to `[sub, key]` for `openai` and `[key]` for the other four, so `openai`'s default method is `sub` and every other provider's is `key`.
- R-4L10-B9DE — for `AuthKey`, `New` constructs a non-nil `agentkit.Provider` when `getenv(EnvKey)` is non-empty, and returns a wrapped `ErrMissingKey` naming the env var (constructing nothing) when it is empty.
- R-4M8W-P143 — for `openai`+`AuthSub`, `New` builds the provider from `subscription.Load(opts.AuthFile)`; a missing or malformed auth file returns a wrapped error naming that path; an `opts.Auth` outside the entry's `Methods` returns a wrapped `ErrAuthUnsupported` naming provider and method.
- R-4NGT-2SUS — `Models(name)` returns exactly agentkit's `ListByProvider(name)` minus entries with `Embedding != nil`, in its sorted order — including, for `openrouter`, the routed GLM entries.
- R-4OOP-GKLH — `Resolve("", m)` derives a cataloged model's home provider; `Resolve("openrouter", "glm-5.2")` yields wire model `z-ai/glm-5.2`; an uncataloged selection reports `ok=false` without error.
- R-4PWL-UCC6 — a cataloged model's entry carries the pricing agentrepl installs on the conversation (`entry.Pricing` non-nil for the priced chat models), and an uncataloged model yields no entry — the seam Decision 3 uses to populate `Conversation.Pricing`.
- R-4R4I-842V — a non-empty `Options.BaseURL` reaches the constructed provider of **any** of the five entries as that package's `WithBaseURL` option (observable via the provider's request URL against a stub server, or the package's documented option plumbing), while an empty `BaseURL` leaves the baked-in default.
