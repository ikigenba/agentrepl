# Decision 4 тАФ CLI flags

**Decision.** The launch surface is deliberately tiny тАФ the config passthrough carries every agentkit setting, so there are no bespoke per-setting flags. Parsing lives in a testable `internal/repl` function over a local `flag.FlagSet` (never the global `flag`), keeping `cmd/agentrepl/main.go` a thin composition root.

```go
package repl

// Options is the parsed launch surface.
type Options struct {
    Config []string // raw "key=value" args, in encounter order
    Raw    bool      // select the raw renderer (default: decorated)
}

// ParseArgs parses argv (excluding program name) into Options.
func ParseArgs(name string, args []string, out io.Writer) (Options, error)
```

| Flag | Meaning |
|------|---------|
| `-c key=value` | config passthrough; **repeatable** (a `flag.Value` slice), applied in encounter order at startup |
| `-raw` | use the raw renderer; default is the decorated transcript |
| `-h` / `-help` | the **self-describing catalog** (Decision 12), not the bare `FlagSet` usage |
| `-V` / `--version` | print the version string and exit `0` тАФ the version reporting owned by **Decision 15**, prescanned in the composition root before parsing (never reaches the config passthrough) |

Provider and model are **not** their own flags тАФ they are `-c provider=тАж -c model=тАж`, keeping the "one config vocabulary, no bespoke flags" promise honest.

- **`-c` is collected raw, validated at apply.** `ParseArgs` only gathers the strings; `Run` builds the `config.Target` and applies each via `config.ParsePair`+`config.Set` тАФ one validation path shared with runtime `/set`.
- **`-h`/`-help` is intercepted into the self-describing catalog.** `ParseArgs` overrides the `FlagSet`'s usage so that `-h`/`-help` writes the Decision 12 catalog to `out` and returns the sentinel `flag.ErrHelp`; the composition root treats that as a clean exit `0` and never starts the loop. The catalog is **credential-blind** тАФ `ParseArgs` builds no `config.Target` and reads no env, so help runs with no keys set.
- **A bad `-c` at launch is fatal** (clear stderr message, exit 1): an impossible initial state should stop, not start surprising. This is the deliberate asymmetry with runtime `/set`, which is non-fatal тАФ and matches the exit taxonomy (startup vs in-loop). This includes the two-tier model rule: a bare `-c model=<uncataloged>` fails at startup (Decision 3).
- **Bare launch is fully usable.** The seeded defaults (`provider=openai`, `model=gpt-5.6-sol`, subscription auth тАФ Decision 3) mean `agentrepl` with no flags drops straight into a working conversation once auth is in place; `-c` pairs override the seed. Credentials are checked lazily (Decision 14), so launching with nothing configured still starts the REPL and directs the operator at the first turn тАФ never a crash.

**Rejected.**
- **`-render decorated|raw`** instead of `-raw` тАФ two modes, decorated default; the bool is simpler (YAGNI).
- **Bespoke `-provider`/`-model`/`-system`/`-temperature` flags** тАФ contradict the single-vocabulary promise.
- **Requiring provider+model at launch** тАФ moot under seeded defaults; a bare start is a fully-selected start.
- **A color flag** тАФ color is auto by terminal detection (Decision 5), not a launch knob.

**Verification.**
- R-EU69-75V4 тАФ `ParseArgs` collects repeated `-c` into `Options.Config` in encounter order.
- R-EWM1-YPCI тАФ `ParseArgs` sets `Raw` from `-raw` (default false); an unknown flag returns an error and writes usage to `out`.
- R-EXTY-CH37 тАФ at startup, `-c` pairs apply in order to the live `Target`, with a later pair overriding an earlier one for the same key.
- R-EZ1U-Q8TW тАФ a launch `-c` with a bad key/value/format is fatal: clear stderr message, exit code 1, and the REPL loop never starts.
