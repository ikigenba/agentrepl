# Decision 12 — The self-describing `--help` catalog

**Decision.** `-h`/`-help` prints a **static, credential-blind catalog** sourced from the provider table (Decision 2) and agentkit's advisory catalog — never from constructed clients or the environment. Rendering lives in `internal/repl` as a pure function; `ParseArgs` invokes it on the help flag (Decision 4) and the composition root exits `0`.

```go
package repl

// WriteHelp renders the static catalog: usage, the launch flags, the defaults
// line, the providers list with per-method auth, the models grouped by provider
// (from agentkit's catalog via catalog.Models), and the two-tier model-rule
// footer. Reads only cat and the agentkit catalog tables — no env, no
// constructed providers — so it is credential-blind.
func WriteHelp(out io.Writer, name string, cat []catalog.Provider)
```

Catalog shape (illustrative — exact bytes pinned by a golden file under `testdata/`):

```
usage: agentrepl [-c key=value ...] [-raw] [-h]

flags:
  -c key=value   set an agentkit config value (repeatable); see config keys via /help at runtime
  -raw           emit the raw, undecorated message stream
  -h, -help      show this catalog and exit

defaults:
  provider=openai   model=gpt-5.6-sol   auth=sub

providers:
  anthropic    auth=key  (ANTHROPIC_API_KEY)
  google       auth=key  (GEMINI_API_KEY)
  openai       auth=key  (OPENAI_API_KEY)
               auth=sub  (auth_file=~/.agentrepl/auth.json; create with /login)
  openrouter   auth=key  (OPENROUTER_API_KEY)
  zai          auth=key  (ZAI_API_KEY)

models:
  anthropic
    claude-fable-5           effort={low|*medium|high|xhigh|max}
    claude-haiku-4-5         thinking_budget=<1024–4096>  (default off)
    claude-opus-4-8          effort={low|medium|*high|xhigh|max}
    ...
  openrouter
    glm-4.6  -> z-ai/glm-4.6   thinking={*on|off}
    glm-5.2  -> z-ai/glm-5.2   effort={high|*max}  (+ toggle)
  ...

a bare -c model=NAME must name a cataloged model (provider is derived from it).
any other model requires -c provider=... too and passes through unvalidated:
no pricing (cost reports 0), reasoning keys unchecked.
```

- **New sections over the prior catalog:** a `defaults:` line stating the seeded launch state (`provider=openai model=gpt-5.6-sol auth=sub`); per-method auth lines in `providers:` (openai gets two — the `sub` line names the default `auth_file` path and points at `/login`); an `openrouter` model section; and a closing **footer** stating the two-tier model rule (bare model must be cataloged; anything else needs an explicit provider and passes through unvalidated — no pricing, reasoning unchecked).
- **Model rows come from `catalog.Models(provider)`** — agentkit's `ListByProvider` with embedding entries filtered (Decision 2) — in its **sorted** order; the renderer does not sort or curate. A **routed** entry (one whose home provider differs, i.e. the GLM models under `openrouter`) renders `model -> wire-slug` before its reasoning clause, making the route visible.
- **Reasoning clauses render from `Entry.Reasoning`** (an `agentkit.ReasoningSpec`) by exactly the established rules, unchanged from the prior design: the native key leads each row (`termToKey(spec.Term)` → `effort`/`thinking_budget`/`thinking_level`/`thinking`); enum/toggle values in `{a|b|c}` with the default `*`-starred in place; ranges as `<Min–Max>` with sentinels and a starred sentinel default, or a trailing `(default <d>)` when the default has no listed slot; the redundant native `Term` dropped from parentheticals with only a residual phrase (`(+ toggle)`) retained; a nil/absent spec renders `(no reasoning control)`. Range bounds keep the established en-dash format the goldens pin.
- **Strictly credential-blind:** `WriteHelp` never calls `Provider.New`, `Getenv`, or `Target.Provider()`. The env-key and auth-file strings are printed as documentation, not presence checks — the live view is `/providers` (Decision 9). The defaults line is the static seed, not current state.

**Rejected.**
- **Rendering only home-provider entries under `openrouter`** (hiding routes) — the routed rows are precisely what documents `Routes`; an empty-looking openrouter section would misread as "openrouter has no models".
- **Listing OpenRouter's full public model space** — thousands of slugs, not agentkit catalog data; the footer's free-flow rule covers them.
- **Showing pricing/context columns** — real data, but the help is a *request* catalog; spend surfaces live via Decision 7.
- **A live defaults line reflecting current env/flags** — help is static and credential-blind by contract.
- *(Carried forward from the prior design, still rejected: the bare `FlagSet` usage; sourcing reasoning from constructed providers; per-provider format branches; hardcoded reasoning text; trailing `default` clauses where a slot exists; folding disable into the enum list; a generic `gen.reasoning` key; a single header naming the key once; keying `termToKey` on `Kind`.)*

**Verification.**
- R-FT8W-Z2U4 — `-h`/`-help` writes the catalog to `out` and exits `0` without starting the REPL loop, and does so with **no** environment variables set and **no** provider constructed (credential-blind).
- R-5873-KWGL — the models section renders, for each provider in table order, exactly `catalog.Models(provider)` (agentkit's sorted list, embedding entries absent), and each routed entry under `openrouter` renders as `model -> wire-slug` (e.g. `glm-5.2  -> z-ai/glm-5.2`) ahead of its reasoning clause (golden-pinned).
- R-5AMW-CFXZ — the catalog carries a `defaults:` section stating `provider=openai`, `model=gpt-5.6-sol`, `auth=sub` (golden-pinned).
- R-5BUS-Q7OO — the providers section lists all five providers with an `auth=key  (<ENV_VAR>)` line each, and openai additionally an `auth=sub` line naming the default auth-file path and `/login` (golden-pinned).
- R-5D2P-3ZFD — the catalog closes with the two-tier footer: a bare model must be cataloged (provider derived), any other model requires an explicit provider and passes through unvalidated with no pricing and unchecked reasoning (golden-pinned).
- R-FVOP-QMBI — each model's accepted-values group is rendered from its `ReasoningSpec` by `Kind`: an enum model shows its `Levels`, a range model shows `Min`–`Max` plus sentinel meanings, a toggle model shows `on`/`off`; a **range** additionally carries a trailing parenthetical for its sentinel meanings and default (golden-pinned across all three kinds).
- R-ODOF-XOTJ — on an enum row and a toggle row, the single value equal to `spec.Default` is prefixed with a `*` inside the `{…}` list and every other value is bare, and the row carries **no** trailing `default …` clause (golden-pinned).
- R-OEWC-BGK8 — an enum/toggle row whose `Term` normalizes exactly to its key prints with **no** trailing parenthetical, while a row whose `Term` carries a phrase beyond the key retains only that residue (GLM 5.x `Term = "effort (+ toggle)"` → `effort={high|*max}  (+ toggle)`).
- R-AFZE-5WGV — a `ReasoningRange` row omits the redundant native `Term` from its trailing parenthetical, carrying only the sentinel meanings and the default: `thinking_budget=<0–24576>  (0=off, -1=*dynamic)` and, with no sentinels, a bare `(default off)` form — the parenthetical never begins with `thinking budget;` (golden-pinned).
- R-APDX-FP3D — a `ReasoningRange` whose `spec.Default` equals a listed sentinel value marks that sentinel in place with a `*` and prints **no** trailing `default …` clause; a range whose default matches no sentinel keeps its trailing `(default <d>)` unchanged (golden-pinned).
- R-6DEO-9TXQ — every model row leads with that model's native reasoning key (`effort`/`thinking_budget`/`thinking_level`/`thinking`, derived from `spec.Term`) followed by its values in traditional CLI syntax — `{a|b|c}` for enum/toggle, `<…>` for a range — so the row is copy-pasteable as `-c <key>=<value>`.
- R-6DEO-KEYS — `termToKey(spec.Term)` for every chat model in the agentkit catalog resolves to one of the four reasoning keys Decision 3 registers, so the catalog never prints a key the config layer would reject as unknown.
- R-FWWM-4E27 — a model whose entry carries no `Reasoning` spec renders a `(no reasoning control)` clause and is not dropped from the listing.
- R-FY4I-I5SW — `WriteHelp` performs no env read and constructs no provider (asserted by passing a provider table whose `New`/`Getenv` would record or panic if called), proving the help path cannot depend on credentials.
