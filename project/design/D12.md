# Decision 12 ‚Äî The self-describing `--help` catalog

**Decision.** `-h`/`-help` prints a **static, credential-blind catalog** sourced entirely from the curated catalog (Decision 2) and agentkit's per-package introspectors ‚Äî never from constructed clients or the environment. It is the launch-time answer to the product's "discover providers, models, and reasoning options without starting a session." Rendering lives in `internal/repl` as a pure function over the catalog and an `io.Writer`; `ParseArgs` invokes it on the help flag (Decision 4) and the composition root exits `0`.

```go
package repl

// WriteHelp renders the static catalog: a one-line usage, the launch flags, the
// providers list, and the models-grouped-by-provider list with each model's
// reasoning shown as its own native config key (`effort`, `thinking_budget`,
// `thinking_level`, or `thinking`, derived from spec.Term) and accepted values
// in traditional CLI syntax (the default value marked inline with a leading `*`).
// It reads only cat (no env, no constructed providers), so it is credential-blind.
func WriteHelp(out io.Writer, name string, cat []catalog.Provider)
```

Catalog shape (illustrative ‚Äî exact bytes pinned by a golden file under `testdata/`):

```
usage: agentrepl [-c key=value ...] [-raw] [-h]

flags:
  -c key=value   set an agentkit config value (repeatable); see config keys via /help at runtime
  -raw           emit the raw, undecorated message stream
  -h, -help      show this catalog and exit

providers:
  anthropic   (ANTHROPIC_API_KEY)
  google      (GEMINI_API_KEY)
  openai      (OPENAI_API_KEY)
  zai         (ZAI_API_KEY)

models:
  anthropic
    claude-opus-4-8     effort={low|medium|*high|xhigh|max}
    claude-haiku-4-5    thinking_budget=<1024‚Äìmax_tokens>          (default off)
  google
    gemini-2.5-flash    thinking_budget=<0‚Äì24576>                  (0=off, -1=*dynamic)
    gemini-3.5-flash    thinking_level={minimal|low|*medium|high}
  openai
    gpt-5.5             effort={none|low|*medium|high|xhigh}
  zai
    glm-5.2             effort={high|*max}                         (+ toggle)
    glm-4.7             thinking={*on|off}
```

- **The model's own native key leads each row.** Every reasoning line is labeled with the config key derived from that model's `spec.Term` ‚Äî `effort`, `thinking_budget`, `thinking_level`, or `thinking` ‚Äî followed by its accepted values in **traditional CLI syntax**: braces `{a|b|c}` for an enumerated choice, angle brackets `<‚Ä¶>` for a free numeric value. A reader copies the exact `-c <key>=<value>` terminology straight off the row, the same way the `providers:`/`models:` section labels telegraph the `provider`/`model` keys.
- **The default is marked in place wherever it has a listed slot.** For the enumerated shapes (`ReasoningEnum` and `ReasoningToggle`) the value that equals `spec.Default` is prefixed with a `*` **inside** the `{‚Ä¶}` list ‚Äî `effort={low|medium|*high|xhigh|max}`, `thinking={*on|off}` ‚Äî and no trailing `default ‚Ä¶` clause is printed. A `ReasoningRange` has no enum slot, but a **sentinel is a slot**: when `spec.Default` equals one of the listed sentinel values the matching sentinel is starred in place (`(0=off, -1=*dynamic)`, `(-1=*dynamic)`) and the trailing `default ‚Ä¶` clause is dropped, exactly as for an enum value. Only a range default with **no** listed slot ‚Äî an `off` with no `0=off` sentinel, or a plain in-range budget ‚Äî keeps a trailing `(default <d>)`. In every shape the `*` marks exactly one token; every other value is bare.
- **The redundant native `Term` is dropped from every row's parenthetical.** The native `Term` for a reasoning model normalizes exactly to the key already leading the row (`thinking budget` ‚Üí `thinking_budget`, `effort` ‚Üí `effort`, `thinking level` ‚Üí `thinking_level`, `thinking` ‚Üí `thinking`), so printing it in the parenthetical is pure repetition and is omitted for all three shapes. For enum/toggle that empties the parenthetical entirely (their default is already starred inline), so the row ends at the `{‚Ä¶}` group; for a range the parenthetical survives carrying only the sentinels and default, term-free. The one exception is a `Term` that carries a phrase **beyond** the key: Z.ai's GLM 5.x reports `Term = "effort (+ toggle)"` (its enum effort levels plus an on/off disable), and that residual `(+ toggle)` is real information the key does not carry, so it is retained as a bare `  (+ toggle)` parenthetical. A row whose `Term` equals its key carries no term text in its parenthetical.
- **The key comes from `spec.Term`, the values group from `spec.Kind`.** A small `termToKey` normalization maps the native term to one of the four registered config keys: lowercase, drop a trailing ` (+ toggle)`, replace the space in `thinking budget`/`thinking level` with `_`. This yields exactly `effort` / `thinking_budget` / `thinking_level` / `thinking` ‚Äî the same four keys Decision 3 registers ‚Äî so the help can never advertise a key the config layer does not accept (golden + a cross-check id below). One render routine then turns a `ReasoningSpec` into its clause ‚Äî the per-model key prefix, then the values group keyed on `spec.Kind` ‚Äî no per-provider formatting:
  - `ReasoningEnum` ‚Üí `<key>={<Levels joined "|", the level equal to spec.Default prefixed "*">}`, then any residual `Term` phrase beyond the key (`(+ toggle)`) as a bare parenthetical, else nothing.
  - `ReasoningRange` ‚Üí `<key>=<<Min>‚Äì<Max>>` then any `Sentinels` and the default in the parenthetical, **without** the redundant `Term`. When `spec.Default` equals a listed sentinel value, that sentinel is starred in place (`v=*meaning`) and **no** trailing `default ‚Ä¶` clause is printed: `(0=off, -1=*dynamic)`, `(-1=*dynamic)`. Otherwise the default is spelled in a trailing clause ‚Äî `(v=meaning, ‚Ä¶; default <d>)`, collapsing to `(default <d>)` when there are no sentinels. The default is rendered from `spec.Default` in its native form (a budget int, a sentinel meaning, or `off` for a disabled default), computed the same way in both the starred and the trailing-clause case.
  - `ReasoningToggle` ‚Üí `<key>={<*-marked on|off>}`, where the value equal to `spec.Default` is starred (an unset/enabled default stars `on`, a disabled default stars `off`); no parenthetical (a toggle's `Term` is always `thinking`, equal to its key).
- **Folding `off` into the enum list was rejected** (see Rejected): a disable-able enum's off state is reachable only through the `thinking` key, never `effort=off`, so listing `off` under the `effort` key would misadvertise a value the config layer would coerce to a bogus level.
- **Reasoning facts come only from `p.Reasoning`** (the Decision 2 inspector). A model whose `ReasoningSpec(id)` returns `false` is rendered with a plain `(no reasoning control)` clause rather than omitted.
- **Default value rendering reuses the same native formatter** the config display side uses (Decision 3): the token that names a model's default (a level, a budget int, a sentinel meaning, or `off`) is computed identically for `--help` and `/get`. `--help` marks that token in place ‚Äî a `*` prefix on the matching enum/toggle value **or the matching range sentinel**, falling back to the `default <token>` tail of a range parenthetical only when the default has no listed slot; `/get` shows it as the current value.
- **Provider order and model grouping follow `catalog.Default()`'s stable order** ‚Äî the catalog owns display order; the renderer does not sort.
- **Strictly credential-blind:** `WriteHelp` takes the catalog and writes text; it never calls `Build`, `Getenv`, or any provider constructor. The env-key name in parens is the catalog's `EnvKey` *string*, printed as documentation, not a presence check ‚Äî that live view is `/providers` (Decision 9).

**Rejected.**
- **The bare `flag.FlagSet` usage** (the prior decision) ‚Äî lists no providers, models, or reasoning; leaves the product's self-describing-help promise unmet.
- **Sourcing reasoning from a constructed provider / `SupportedReasoning()` on a handle** ‚Äî would need a key just to print static metadata; the credential-blind package-level inspector exists precisely to avoid this.
- **Per-provider format branches** ‚Äî reintroduces the provider knowledge the thin-consumer principle forbids; a single `Kind`-keyed routine renders all three shapes.
- **Hardcoding the reasoning text in agentrepl** ‚Äî drifts from agentkit the moment a model's vocabulary changes; reading `ReasoningSpec` at print time keeps display and acceptance from one source.
- **Spelling the default out in a trailing `(‚Ä¶; default <x>)` clause** (the prior design) ‚Äî for the enumerated shapes the default is one of the very values already listed in the `{‚Ä¶}` group, so a separate clause restates it at a distance; marking that value in place with a `*` puts the default where the reader is already looking and drops a redundant parenthetical. Kept only for a `ReasoningRange` default with **no** listed slot ‚Äî an `off` with no matching sentinel, or a plain in-range budget; a range default that coincides with a listed sentinel is starred in place like an enum value, since the sentinel is exactly such a slot.
- **Keeping the native `Term` in the range parenthetical** (the prior design) ‚Äî a range row already leads with `thinking_budget=`, and its `Term` (`thinking budget`) normalizes to exactly that key, so printing `(thinking budget; ‚Ä¶)` restates the key one glyph away from itself. Dropping it applies the same redundant-term rule the enum/toggle rows already follow, leaving the range parenthetical to carry only what the key does not: the sentinel meanings and the default. An `off`-marked default with no matching sentinel (e.g. `claude-haiku-4-5`) still needs the parenthetical for its `default off`, so the parenthetical is emptied only for enum/toggle, never for range.
- **Folding the disable state into the enum values list** (e.g. `effort={high|*max|off}` for a `CanDisable` enum) ‚Äî the row leads with the `effort` key, but `-c effort=off` does **not** disable: Decision 3 coerces `effort` verbatim to `Level("off")`, a bogus level that is warned-and-defaulted, never a disable. A disable-able enum's off state is reachable only via `thinking=off` (`DisableReasoning()`). Listing `off` under `effort` would therefore advertise a value the row's own key rejects, breaking the copy-paste promise; and because `CanDisable` is near-universal across the catalog yet unsurfaced today, folding it in would newly clutter almost every row. The disable path stays reachable through the `thinking` key, and the only disable hint shown is agentkit's own `(+ toggle)` `Term` residue.
- **A single generic `gen.reasoning` key for every model** (the prior design) ‚Äî a synthetic common interface over a setting that is natively per-model; removed in favor of passthrough native keys. The `gen.`/`retry.`/`zai.` prefixes went with it: they never routed anything (`Set` is one flat `map[string]field` lookup), so they were pure decoration.
- **A single header line naming the key once** (`models:  (set with -c <key>=<value>)`) ‚Äî forces the reader to assemble the key from a distant header and the values from the row; each row should be independently copy-pasteable, carrying its own native key.
- **A `termToKey` that keys on `spec.Kind` instead of `spec.Term`** ‚Äî `Kind=Enum` is shared by `effort` (Anthropic/OpenAI/Z.ai) and `thinking_level` (Gemini), so the kind alone cannot pick the key; the native term is the only unambiguous source.

**Verification.**
- R-FT8W-Z2U4 ‚Äî `-h`/`-help` writes the catalog to `out` and exits `0` without starting the REPL loop, and does so with **no** environment variables set and **no** provider constructed (credential-blind).
- R-FUGT-CUKT ‚Äî the catalog lists every provider from `catalog.Default()` and, grouped under each, every curated model id, in `Default()`'s order.
- R-FVOP-QMBI ‚Äî each model's accepted-values group is rendered from its `ReasoningSpec` by `Kind`: an enum model shows its `Levels`, a range model shows `Min`‚Äì`Max` plus sentinel meanings, a toggle model shows `on`/`off`; a **range** additionally carries a trailing parenthetical for its sentinel meanings and default (golden-pinned across all three kinds).
- R-ODOF-XOTJ ‚Äî on an enum row and a toggle row, the single value equal to `spec.Default` is prefixed with a `*` inside the `{‚Ä¶}` list and every other value is bare, and the row carries **no** trailing `default ‚Ä¶` clause (e.g. `effort={low|medium|*high|xhigh|max}` for a `high` default, `thinking={*on|off}` for an enabled default) (golden-pinned).
- R-OEWC-BGK8 ‚Äî an enum/toggle row whose `Term` normalizes exactly to its key prints with **no** trailing parenthetical, while a row whose `Term` carries a phrase beyond the key retains only that residue (Z.ai GLM 5.x `Term = "effort (+ toggle)"` ‚Üí `effort={high|*max}  (+ toggle)`).
- R-AFZE-5WGV ‚Äî a `ReasoningRange` row omits the redundant native `Term` from its trailing parenthetical (whose normalization equals the row's leading key), carrying only the sentinel meanings and the default: `thinking_budget=<0‚Äì24576>  (0=off, -1=*dynamic)` and, with no sentinels, `thinking_budget=<1024‚Äìmax_tokens>  (default off)` ‚Äî the parenthetical never begins with `thinking budget;` (golden-pinned).
- R-6DEO-9TXQ ‚Äî every model row leads with that model's native reasoning key (`effort`/`thinking_budget`/`thinking_level`/`thinking`, derived from `spec.Term`) followed by its values in traditional CLI syntax ‚Äî `{a|b|c}` for enum/toggle, `<‚Ä¶>` for a range ‚Äî so the row is copy-pasteable as `-c <key>=<value>`.
- R-6DEO-KEYS ‚Äî `termToKey(spec.Term)` for every supported model resolves to one of the four reasoning keys Decision 3 registers (`effort`/`thinking_budget`/`thinking_level`/`thinking`), so the catalog never prints a key the config layer would reject as unknown.
- R-FWWM-4E27 ‚Äî a model whose inspector returns `ReasoningSpec(id) == (_, false)` renders a `(no reasoning control)` clause and is not dropped from the listing.
- R-APDX-FP3D ‚Äî a `ReasoningRange` whose `spec.Default` equals a listed sentinel value marks that sentinel in place with a `*` and prints **no** trailing `default ‚Ä¶` clause (`thinking_budget=<0‚Äì24576>  (0=off, -1=*dynamic)`, `thinking_budget=<128‚Äì32768>  (-1=*dynamic)`); a range whose default matches no sentinel ‚Äî a slot-less `off` or a plain in-range budget ‚Äî keeps its trailing `(default <d>)` unchanged (`thinking_budget=<1024‚Äìmax_tokens>  (default off)`) (golden-pinned).
- R-FY4I-I5SW ‚Äî `WriteHelp` performs no env read and constructs no provider (asserted by passing a catalog whose `New`/`Getenv` would record or panic if called), proving the help path cannot depend on credentials.
