# Decision 14 — Subscription auth, `/login`, and lazy credential failure

**Decision.** OpenAI subscription auth is wired through agentkit's `openai/subscription` package against an **agentrepl-owned auth file**, defaulting to `~/.agentrepl/auth.json`; the file is created by the `/login` command — **agentrepl owns the entire interactive surface** (all printing, prompting, and reading) over agentkit's value-in/value-out OAuth flow (`BeginLogin`/`Flow.Complete`, agentkit ≥ v0.4.1) — and consumed by the lazy provider build (Decisions 2/3). Credentials are never checked at startup — a build failure surfaces at the first turn that needs the provider, as a **directive** message naming the remedies.

```go
package repl

// LoginFlow is the REPL's view of one in-flight OAuth login. agentkit's
// *subscription.Flow satisfies it via a tiny adapter in the composition
// root; tests inject a fake.
type LoginFlow interface {
    AuthorizeURL() string
    Complete(ctx context.Context, path, pastedRedirectURL string) error
}

// Deps carries the login seam; the composition root binds it to
// subscription.BeginLogin (adapted), tests to a fake constructor.
type Deps struct {
    // …existing fields (IO, Getenv, Now, LogDir)…
    AuthFile   string // default subscription auth file path: <home>/.agentrepl/auth.json, resolved by the composition root
    BeginLogin func() (LoginFlow, error)
}
```

- **The default auth file is agentrepl's own.** `~/.agentrepl/auth.json` (resolved to an absolute path by the composition root and passed through `Deps.AuthFile` into `config.NewTarget`) deliberately avoids the *live* `~/.codex/auth.json`: agentkit's `Store` rewrites the file on token refresh, and sharing Codex CLI's file risks refresh-token-rotation contention (research §4). An operator with an existing Codex login opts in explicitly with `-c auth_file=~/.codex/auth.json` (or a copy).
- **`/login` owns the whole interaction; agentkit supplies only values.** The handler calls `Deps.BeginLogin()`, then prints the authorize URL with explicit instructions: open the URL in any browser, authorize, the browser will land on a dead `http://localhost:1455/...` page (nothing listens there — expected), copy the **full URL from the address bar**, and paste it at the prompt. It then reads **exactly one more line through the REPL's own input** (the same scanner the loop uses — stdin is never handed to a second reader). The line, trimmed: **empty (or EOF) cancels** — a "login cancelled" notice, `Complete` never called (EOF then winds the loop down as usual); **anything else is the paste**, passed verbatim to `flow.Complete(ctx, target.AuthFile, line)` — no slash-command interception inside this one-line window. On nil error: success notice, and the lazily built provider is invalidated so the **next turn picks the new credentials up without a restart**. On error (agentkit's clear "expected the full redirect URL" / state-mismatch messages included): a non-fatal rendered error, loop continues. `/login` is meaningful regardless of the currently selected provider (it just writes the file).
- **Lazy failure is directive, not generic.** When the turn driver's `target.Provider()` call fails (Decision 9), the rendered error tells the operator exactly what to do, keyed to the failed method:
  - `sub`: the auth-file problem and path, then the three remedies — run `/login`, or `/set auth key` (with `OPENAI_API_KEY`), or point `auth_file` at an existing Codex login.
  - `key`: the missing env var by name (the established `ErrMissingKey` rendering).
  The turn is not sent; the loop continues (Decision 11's resilience invariant).
- **Startup never touches credentials.** Seeding the defaults constructs nothing (Decision 3); `agentrepl` on a machine with no keys and no auth file starts, prints its prompt, and answers every slash-command — only a turn attempt surfaces the directive.
- **Token refresh is agentkit's.** The `Store` auto-refreshes and atomically rewrites the file; agentrepl never parses, prints, or edits the file's contents itself — it only passes the path.

**Rejected.**
- **A startup credential check** — would make the bare out-of-box launch (subscription default, no file yet) fail before `/login` could ever run.
- **Defaulting `auth_file` to `~/.codex/auth.json`** — the live-file contention risk (two writers rotating one refresh token) is real; the safer agentrepl-owned default costs one `/login` or one explicit `-c auth_file=…`.
- **A `-login` launch flag** — the flow is interactive (paste a code back); it belongs in the REPL where the IO already lives.
- **agentrepl implementing its own OAuth flow or token refresh** — agentkit ships both (`BeginLogin`/`Flow.Complete`, `Store.Token`); reimplementation contradicts the thin-consumer principle.
- **Handing the REPL's In/Out to an agentkit-owned interactive login** — the original wiring (agentkit v0.4.0's `Login(ctx, path, LoginIO)`): the library blocked on stdin with no prompt of its own, the REPL lost its dispatcher for the duration, and the operator, unprompted, fed it a stray line that surfaced as a bogus state-mismatch error. agentkit v0.4.1 deleted that surface; presentation is the consumer's job.
- **Intercepting slash commands inside the paste window** — a one-line modal window doesn't warrant a second command grammar; empty-line cancel is the escape hatch.

**Verification.**
- R-LRTQ-4X56 — `/login` calls the injected `BeginLogin`, prints the flow's authorize URL together with instructions naming the dead-`localhost:1455`-page/copy-the-address-bar/paste-back steps and a paste prompt, reads exactly one line through the REPL's own input, passes it trimmed to `Complete` with the **current** `auth_file` path, and on nil error renders a success notice and invalidates the cached provider so the next build reads the new file.
- R-LT1M-IOVV — an empty paste line (or EOF at the paste prompt) cancels: a cancellation notice is rendered, `Complete` is never called, and the loop accepts the next input (EOF then terminates the loop as it normally does).
- R-LU9I-WGMK — a `Complete` error (and equally a `BeginLogin` error) is rendered as a non-fatal error preserving the underlying message, the provider cache is **not** invalidated, and the loop accepts the next input.
- R-5FIH-VIWR — a turn whose provider build fails under subscription auth renders a directive error naming the auth-file path, `/login`, and the `auth=key` alternative, does **not** call `Send`, and leaves the loop accepting the next input; a key-method build failure renders the missing env var by name with the same non-fatal behavior.
- R-5GQE-9ANG — with no credentials and no auth file present anywhere, bare startup succeeds: the REPL reaches its prompt and slash-commands (`/help`, `/providers`, `/dump`) answer normally; no credential read or provider construction happens before the first turn attempt.
- R-5HYA-N2E5 — round-trip: after a (faked) login seam writes a valid subscription auth file at the configured path, the next turn's lazy build succeeds against that file in the same session — no restart — proving the invalidate-on-login and `subscription.Load` wiring.
