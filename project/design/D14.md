# Decision 14 — Subscription auth, the external login, and lazy credential failure

**Decision.** OpenAI subscription auth is wired through agentkit's `openai/subscription` package (agentkit ≥ v0.5.0) against an **agentrepl-owned auth file**, defaulting to `~/.agentrepl/auth.json`. The file's shape is agentkit's contract — the raw OAuth token-endpoint response — and **agentrepl never creates it**: the operator produces it outside the REPL with the standalone `oauth-login` CLI, redirecting its stdout to the auth file. agentrepl's whole job is to consume the file lazily (Decisions 2/3) and, when that fails, to tell the operator exactly how to produce it. There is no `/login` command, no login seam, and no login state in the REPL.

```go
package repl

// Deps carries no login machinery. The auth-file path is the only
// subscription-related dependency.
type Deps struct {
    // …existing fields (IO, Getenv, Now, LogDir)…
    AuthFile string // default subscription auth file path: <home>/.agentrepl/auth.json, resolved by the composition root
}
```

- **The default auth file is agentrepl's own.** `~/.agentrepl/auth.json` (resolved to an absolute path by the composition root and passed through `Deps.AuthFile` into `config.NewTarget`). `auth_file` remains a config key: the operator points it anywhere; whatever raw-token-response file `oauth-login` produced there is what gets loaded.
- **The login command is a printed artifact, not a feature.** One unexported constant assembles the ready-to-run invocation from the resolved auth-file path, as a **single line** (paste-able as-is):

      oauth-login --auth-url https://auth.openai.com/oauth/authorize --token-url https://auth.openai.com/oauth/token --client-id app_EMoamEEZ73f0CkXaXp7hrann --scope "openid profile email offline_access" --port 1455 --callback-path /auth/callback > <auth_file>

  The flag values mirror oauth-login's own documented OpenAI example (`--port 1455 --callback-path /auth/callback` match OpenAI's registered redirect URI); `<auth_file>` is the current `auth_file` value. agentrepl treats the string as opaque guidance — it never runs the command, never parses the file, never touches OAuth.
- **Lazy failure is directive, not generic.** When the turn driver's `target.Provider()` call fails (Decision 9), the rendered error tells the operator exactly what to do, keyed to the failed method:
  - `sub`: the auth-file problem and path, then the remedies — the full single-line `oauth-login` command above (run it in another terminal, then just take the next turn), or `/set auth key` (with `OPENAI_API_KEY`), or point `auth_file` at an existing file produced by `oauth-login`.
  - `key`: the missing env var by name (the established `ErrMissingKey` rendering).
  The turn is not sent; the loop continues (Decision 11's resilience invariant).
- **Pickup needs no restart and no command.** Provider construction is lazy and retried per turn (Decision 3): a turn taken after the operator has run `oauth-login` externally builds against the now-present file. Nothing is cached across a failed build, so there is nothing to invalidate.
- **Startup never touches credentials.** Seeding the defaults constructs nothing (Decision 3); `agentrepl` on a machine with no keys and no auth file starts, prints its prompt, and answers every slash-command — only a turn attempt surfaces the directive.
- **Token refresh is agentkit's.** The `Store` auto-refreshes and atomically rewrites the file; agentrepl never parses, prints, or edits the file's contents itself — it only passes the path.

**Rejected.**
- **The in-REPL `/login` flow** — the prior design (agentkit v0.4.1's value-in/value-out `BeginLogin`/`Flow.Complete`, agentrepl owning the paste interaction). Its manual-paste UX dead-ended on a browser page nothing served, and its token-response parsing failed against the real endpoint; the standalone `oauth-login` tool does the whole handshake properly (served callback, browser success page) for any OAuth provider. agentkit v0.5.0 deleted the flow surface; agentrepl follows.
- **Shelling out to `oauth-login` from a `/login` wrapper** — operator-rejected in favor of full removal: the REPL holds zero login machinery, not even an exec; the printed command is the entire surface.
- **Defaulting `auth_file` to `~/.codex/auth.json`** — doubly wrong now: the live-file contention risk (two writers rotating one refresh token), and agentkit v0.5.0 no longer reads the codex wrapper shape at all.
- **A startup credential check** — would make the bare out-of-box launch (subscription default, no file yet) fail before the operator ever saw the directive.
- **agentrepl implementing OAuth or token refresh itself** — `oauth-login` owns the handshake, agentkit owns refresh; reimplementation contradicts the thin-consumer principle.

**Verification.**
- R-E6D8-E6BS — a turn whose provider build fails under subscription auth renders a directive error naming the auth-file path and containing the full single-line `oauth-login` invocation — `--client-id app_EMoamEEZ73f0CkXaXp7hrann`, `--port 1455`, `--callback-path /auth/callback`, and a trailing `> <auth_file>` redirect carrying the **current** `auth_file` value (a non-default `auth_file` appears verbatim) — plus the `/set auth key` alternative; the command occupies one line (no line breaks within it), `Send` is not called, and the loop accepts the next input; a key-method build failure renders the missing env var by name with the same non-fatal behavior.
- R-5GQE-9ANG — with no credentials and no auth file present anywhere, bare startup succeeds: the REPL reaches its prompt and slash-commands (`/help`, `/providers`, `/dump`) answer normally; no credential read or provider construction happens before the first turn attempt.
- R-E7L4-RY2H — a subscription turn that fails for a missing auth file, followed by the file appearing at the configured path (written externally, valid raw-token-response shape), is followed by a next turn in the same session that lazily builds against that file and sends — no restart, no command in between — proving per-turn lazy construction and the `subscription.Load` wiring.
