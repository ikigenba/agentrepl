# Phase 26 — `--help` drops the redundant native term from range rows

*Realizes design Decision 12 (amended: the `ReasoningRange` parenthetical omits the native `Term`, which normalizes to the key already leading the row, and carries only the sentinels and default). Depends on Phase 25 (`WriteHelp`, its `Kind`-keyed render routine, and its golden).*

Rework the `ReasoningRange` branch of the render routine inside `repl.WriteHelp` so the trailing parenthetical no longer restates the native `Term`. A range row leads with `thinking_budget=`, and its `Term` (`thinking budget`) normalizes via `termToKey` to exactly that key, so it is dropped as redundant — the same rule the enum/toggle rows already apply. The parenthetical then carries only what the key does not: any `Sentinels` (`0=off, -1=dynamic`) and the native default (`default dynamic`), e.g. `thinking_budget=<0–24576>  (0=off, -1=dynamic; default dynamic)`; a range with no sentinels collapses to `(default off)` (e.g. `claude-haiku-4-5`). The enum/toggle branches, the `ReasoningRange` value syntax `<Min>–<Max>`, the shared native default formatter (Decision 3), `termToKey`, the `(no reasoning control)` fallback, the credential-blind contract, and `Default()`-order grouping are all unchanged. The change is observable only in the rendered bytes, so the `WriteHelp` golden under `testdata/` is regenerated to the term-free range rows.

**Done when:** the following is covered by clearly-named tests (the regenerated `WriteHelp` golden plus a focused assertion on the range rows) and the suite is green:
- R-AFZE-5WGV — a `ReasoningRange` row omits the redundant native `Term` from its trailing parenthetical, carrying only the sentinel meanings and the default (`thinking_budget=<0–24576>  (0=off, -1=dynamic; default dynamic)`, and with no sentinels `thinking_budget=<1024–max_tokens>  (default off)`); the parenthetical never begins with `thinking budget;`.
