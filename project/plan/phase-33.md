# Phase 33 — agentkit v0.5.0 pin + `/login` removed for the external `oauth-login` flow

*Realizes design Decision 14 (subscription auth — the external-login rework: R-E6D8-E6BS, R-E7L4-RY2H), Decision 9 (command set — `/login` removal), and Decision 12 (help catalog — the `auth=sub` line re-pin: R-5BUS-Q7OO). Depends on Phase 32, and on agentkit's published `v0.5.0` tag (its `openai/subscription` consumes the raw token-response auth file and ships no login flow).*

The `go.mod` pin moves to `github.com/ikigenba/agentkit v0.5.0` and every trace of in-REPL login is deleted per the rewritten D14: the `/login` command leaves the dispatch table and `/help`; the `LoginFlow` interface, `Deps.BeginLogin`, the composition-root adapter over `subscription.BeginLogin`, and the paste-window input machinery are all removed (no compatibility shims). The subscription lazy-failure directive is reworked to print, on one line, the full ready-to-run `oauth-login` invocation from D14 — client id `app_EMoamEEZ73f0CkXaXp7hrann`, port `1455`, callback path `/auth/callback`, stdout redirected to the **current** `auth_file` — plus the `/set auth key` alternative and the `auth_file` pointer. Pickup relies on the existing per-turn lazy build: no invalidation hook remains because nothing caches a failed build. The `--help` catalog's `auth=sub` line now points at `oauth-login` instead of `/login` (goldens updated). Tests for the retired login-interaction ids (R-LRTQ-4X56, R-LT1M-IOVV, R-LU9I-WGMK) and the superseded directive/round-trip ids (R-5FIH-VIWR, R-5HYA-N2E5) and `/login`-dispatch id (R-56Z7-74PW) are deleted with the behavior; the surviving startup-laziness behavior stays covered by its existing R-5GQE-9ANG test.
