# Phase 25 — `--help` marks each enum/toggle default inline with `*`

*Realizes design Decision 12 (amended: the enumerated shapes mark their default value in place with a leading `*` and drop the trailing `default …` clause and the now-redundant `Term` parenthetical; the range shape is untouched). Depends on Phase 17 (`WriteHelp`, `termToKey`, and its golden).*

Rework the render routine inside `repl.WriteHelp` so the `ReasoningEnum` and `ReasoningToggle` rows show their default **inside** the `{…}` values group rather than in a trailing clause. For an enum, the single level equal to `spec.Default` is prefixed with `*` and every other level is bare (`effort={low|medium|*high|xhigh|max}`); for a toggle, the `on`/`off` value equal to `spec.Default` is starred (an unset/enabled default stars `on`, a disabled default stars `off` — `thinking={*on|off}`). The trailing `(…; default <x>)` parenthetical is dropped for these two shapes: the native `Term` normalizes to the key already leading the row, so it is pure repetition and omitted, **except** a residual `Term` phrase beyond the key — Z.ai GLM 5.x's `"effort (+ toggle)"` — which is retained as a bare `  (+ toggle)` (`effort={high|*max}  (+ toggle)`). The `ReasoningRange` branch is unchanged: it keeps `<Min>–<Max>` plus sentinels and its native default in the trailing parenthetical, since a continuous range has no listed slot to mark. The default token is still computed by the shared native formatter (Decision 3), so `--help` and `/get` name a model's default identically; the routine stays a single `Kind`-keyed function with no per-provider branches; `termToKey`, the `(no reasoning control)` fallback, the credential-blind contract, and `Default()`-order grouping are unchanged. The change is observable only in the rendered bytes, so the `WriteHelp` golden under `testdata/` is regenerated to the new row shape.

**Done when:** the following are covered by clearly-named tests (the regenerated `WriteHelp` golden plus a focused assertion on the enum/toggle rows) and the suite is green:
- R-ODOF-XOTJ — on an enum row and a toggle row, the single value equal to `spec.Default` is prefixed with `*` inside the `{…}` list and every other value is bare, and no trailing `default …` clause is printed (`effort={low|medium|*high|xhigh|max}`, `thinking={*on|off}`); a range row still renders its default in the parenthetical.
- R-OEWC-BGK8 — an enum/toggle row whose `Term` normalizes exactly to its key prints with no trailing parenthetical, while one whose `Term` carries a phrase beyond the key retains only that residue (GLM 5.x → `effort={high|*max}  (+ toggle)`); range rows keep the full `(<Term>; sentinels; default <d>)` parenthetical.
