# Phase 32 — agentkit v0.4.1 pin + REPL-owned `/login` interaction

*Realizes design Decision 14 (subscription auth — the login-interaction slice: R-LRTQ-4X56, R-LT1M-IOVV, R-LU9I-WGMK). Depends on Phase 31, and on agentkit's published `v0.4.1` tag (its `BeginLogin`/`Flow.Complete` surface replaces the deleted `Login`/`LoginIO`).*

The `go.mod` pin moves to `github.com/ikigenba/agentkit v0.4.1` and the `/login` command becomes fully REPL-owned per the rewritten D14: `internal/repl` declares the `LoginFlow` interface and `Deps.BeginLogin func() (LoginFlow, error)` (replacing `Deps.Login`); `cmd/agentrepl/main.go` binds it to `subscription.BeginLogin` through a small adapter. The handler prints the authorize URL with the dead-`localhost:1455`/copy-the-address-bar/paste-back instructions and a paste prompt, reads exactly one more line through the REPL's own scanner (stdin is never handed to a second reader), treats an empty line or EOF as cancellation (no `Complete` call), and passes any other trimmed line to `flow.Complete` with the current `auth_file`; success renders a notice and invalidates the cached provider, errors render non-fatally and the loop continues. No `subscription.LoginIO` reference survives anywhere in the module.

**Done when:** R-LRTQ-4X56, R-LT1M-IOVV, and R-LU9I-WGMK are covered by clearly-named tagged tests; the retired tag is gone (`grep -rn "R-5EAL-HR62\|LoginIO" --include='*.go' .` returns no matches); `grep -n "agentkit v0.4.1" go.mod` matches; and the green bar holds (`go build ./...`, `go vet ./...`, `go test ./...` all exit 0, `gofmt -l .` prints nothing).
