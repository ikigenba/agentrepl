# Phase 18 — Decorated input prompt & per-turn-report removal

*Realizes design Decision 5 (amended: the `Prompt()`/`Input(text)` interface split, the `tty`-gated `you ›` affordance, the decorated `Input`/`Usage` no-ops, the removed `─` separator, and `NewDecorated(out, color, tty)`), Decision 7 (amended: per-turn spend is raw-only — the decorated view shows only the cumulative summary), and Decision 9 (the loop draws `Prompt()` before every read; `/render` reconstructs with `state.io.IsTTY`). Depends on Phase 6 (render), Phase 7a and 7b (the loop, command dispatch, and turn driver), and Phase 8 (the composition root).*

This is an **atomic interface change** spanning `internal/render`, `internal/repl`, and `cmd/agentrepl/main.go`: the moment `render.Renderer` swaps its single `Prompt(text)` method for `Prompt()` + `Input(text)` and `NewDecorated` gains a `tty` parameter, the repl loop and the composition root stop compiling, so all three move together and the build is green only at phase end (the Phase 11 precedent — no transitional shim). End state in `internal/render`: the `Renderer` interface declares `Prompt()` (draw the input affordance) and `Input(text string)` (record the entered message); `NewDecorated(out, color, tty)` carries `tty` independent of `color`. The **decorated** impl draws `you ›` (bold, color-gated, no trailing newline) from `Prompt()` only when `tty` and nothing otherwise, makes `Input(text)` a no-op (the terminal already echoed the line — no re-render), makes `Usage(...)` a no-op (per-turn spend is raw-only), and drops the `─` separator from `MessageDone`; its cumulative `Summary` is unchanged. The **raw** impl is behaviorally unchanged — `Prompt()` is a no-op, `Input(text)` emits the `{"type":"prompt","text":…}` entry, and the per-turn usage line still fires. In `internal/repl`: the `Run` loop calls `Renderer.Prompt()` before awaiting each input line (so the prompt precedes turns, commands, and empty lines alike), the turn driver calls `Input(text)` in place of the old `Prompt(text)`, and `/render decorated` reconstructs via `state.io.IsTTY`. In the composition root: `NewDecorated` is wired with the existing `IsTTY` alongside `color`. Because the interface changes, the existing render goldens (decorated transcript with the old echoed prompt, the `─` separator, and the per-turn usage line) and the repl-level tests that drove the old `Prompt(text)` path are regenerated/updated to the new shape as part of staying green — the Phase 10/11 "update call sites and goldens" pattern.

**Done when:** these are covered by clearly-named tests (render goldens for the tty and non-tty decorated output and the unchanged raw output; repl-level captured-stdout assertions for the prompt placement) and the suite is green:
- Decision 5 (amended) — R-JFBW-TYU8 (the loop calls `Prompt()` before every read — turn, command, empty line; decorated draws `you ›` with no trailing newline when `tty` and nothing when not, and `Input` never echoes the entered text), R-JGJT-7QKX (decorated `Usage` prints nothing while the same driver call in raw emits the per-turn line), and the reworded R-LL9K-SKDQ (decorated kinds with **no** `─` separator, regenerated golden) and R-LOX9-XVLT (raw prompt entry from `Input`, nothing from `Prompt()`).
- Decision 7 (amended) — the per-turn ids whose wording is now scoped to the raw view, R-ONJY-6PJG and R-OORU-KHA5, still pass unchanged (raw per-turn behavior is untouched); the decorated cumulative-only path remains proven by R-OSFJ-PSI8 / R-OUVC-HBZM (Phases 7b/8).
